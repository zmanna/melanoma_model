## Documentation Principles

- Be explicit that this is an academic prototype, not a clinical tool.
- Separate team-level work from individual portfolio framing.
- Document model and deployment risks plainly.
- Prefer accurate known limitations over polished but unsupported claims.
- Keep dataset, preprocessing, threshold, and evaluation assumptions traceable.
